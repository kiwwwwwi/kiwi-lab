# Kubernetes 핵심 개념

쿠버네티스의 두 가지 핵심 개념.

## API — 애플리케이션 정의

원하는 상태를 선언하는 수단. YAML 매니페스트로 작성하고 API Server에 전달한다.

```yaml
apiVersion: apps/v1
kind: Deployment
spec:
  replicas: 3
```

## 클러스터 — 애플리케이션 실행 환경

선언한 상태가 실제로 동작하는 곳. 노드들의 집합.

```
노드 1 → 파드 실행 중
노드 2 → 파드 실행 중
노드 3 → 파드 실행 중
```

## 둘의 관계

API와 클러스터는 분리되어 있다.

```
개발자
  → YAML 작성 (API로 원하는 상태 선언)
  → kubectl apply → API Server
  → 쿠버네티스가 알아서 클러스터에 실현
```

개발자는 "파드 3개 띄워줘"라고 선언만 하면 되고, 어느 노드에서 실행할지는 신경 쓰지 않아도 된다. 쿠버네티스가 클러스터 상태를 보고 알아서 결정한다.

- **API** = 의도를 표현하는 수단
- **클러스터** = 의도가 실현되는 환경
