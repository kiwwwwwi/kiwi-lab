## Kubernetes

**[Service](../kubernetes/service.md)**
파드는 재시작마다 IP가 바뀌는데, Service가 고정 진입점(ClusterIP + DNS)을 제공해서 클라이언트가 파드 교체를 신경 쓰지 않아도 됨. iptables가 ClusterIP → 파드 IP 변환을 커널 레벨에서 처리.

**[Namespace](../kubernetes/namespace.md)**
리소스를 논리적으로 격리하는 공간. 이름 충돌 방지, 권한 분리, 리소스 쿼터 설정에 활용. 다른 네임스페이스 리소스는 `kafka.kiwi-lab.svc.cluster.local` 처럼 풀네임으로 접근.

**[Operator & Controller](../kubernetes/operator-controller.md)**
Controller는 현재 상태를 원하는 상태로 맞추는 reconciliation 루프. Operator는 Controller + CRD + 도메인 지식의 묶음으로, Kafka 파티션 재분배나 롤링 업그레이드처럼 쿠버네티스가 모르는 복잡한 도메인 지식까지 처리.

**[API Server](../kubernetes/api-server.md)**
클러스터의 모든 요청이 통과하는 중앙 진입점. 구성 요소들은 서로 직접 통신하지 않고 API Server를 거침. `kubectl`은 API Server에 REST 요청을 보내는 HTTP 클라이언트.

**[apiVersion](../kubernetes/api-version.md)**
`apiVersion + kind` 조합으로 리소스 종류를 판단. API 그룹은 같은 `kind` 이름 충돌을 방지하는 논리적 분류. `v1`은 안정 버전으로 하위 호환성 보장, `v1beta1`은 스펙이 바뀔 수 있는 베타.

**[Ingress](../kubernetes/ingress.md)**
외부 HTTP 요청을 클러스터 내부 Service로 라우팅하는 L7 규칙. NodePort(L4)는 IP:포트만 보고, Ingress(L7)는 Host 헤더와 Path까지 볼 수 있어 도메인 기반 라우팅 가능. 실제 트래픽은 Ingress Controller(nginx, traefik 등)가 처리.
